# GitHub Profile README Design

## Goal

Create a playful GitHub profile README for Tacoo that makes the tech stack the main attraction while still feeling friendly and personal.

## Profile Direction

Use a "playful developer card" style:

- Centered greeting introducing Tacoo.
- Short tagline about building with Python, JavaScript, React, Node.js, and web tools.
- Lightweight visual flair through Markdown-compatible badges and GitHub stat cards.
- No fake contact links or social links unless Tacoo provides real destinations later.

## README Structure

1. Header
   - Centered title: "Hi, I'm Tacoo"
   - Short playful subtitle focused on learning, building, and experimenting with web projects.

2. Animated typing banner
   - Use `readme-typing-svg` to show rotating lines such as:
     - Python tinkerer
     - React explorer
     - Node.js builder
     - Linux learner

3. Tech stack
   - Group badges into readable rows.
   - Languages: Python, JavaScript, HTML5, CSS3
   - Frontend: React
   - Backend: Node.js
   - Tools and platform: Git, VS Code, Linux, Hostinger

4. AI toolkit
   - Add a supporting row for AI tools Tacoo uses:
     - ChatGPT
     - Claude
     - GitHub Copilot
     - Cursor
     - Codex

5. Current focus
   - Include a compact list with personality:
     - Building web projects
     - Practicing React and Node.js
     - Improving Linux workflow
     - Experimenting with hosting

6. GitHub stats
   - Use username `takotaks`.
   - Include compact cards for GitHub stats and top languages.
   - Include streak stats only if the final README remains visually clean.

7. Closing
   - Add a simple line inviting visitors to check out projects or follow along.

## Constraints

- Keep the README Markdown-only.
- Do not add a build system, package dependencies, or generated assets.
- Keep remote image widgets limited to common README services such as Shields.io, GitHub Readme Stats, and Readme Typing SVG.
- Avoid clutter: badge rows should support the profile, not overwhelm it.
